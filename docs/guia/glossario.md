## Termos

- **API REST** é uma interface de programação de aplicações que segue conformidade com as restrições da arquitetura REST. A sigla REST significa Representational State Transfer (Transferência Representacional de Estado, em português). Em outras palavras, uma API REST é uma API que utiliza o protocolo HTTP para realizar requisições e transferir dados. Ela é utilizada para integrar sistemas e permitir que diferentes aplicações possam se comunicar entre si.
- **Path** Em programação, PATH é um termo que se refere ao caminho de um arquivo ou diretório. O PATH é uma variável de ambiente que contém uma lista de diretórios separados por dois pontos.
- **Request body** é o dado que é enviado para o servidor como parte de uma requisição HTTP. Ele é geralmente usado em requisições `POST` e `PUT` para enviar dados para o servidor.
- **Response body** é a parte da resposta HTTP que contém o conteúdo da resposta. Ele é geralmente usado para enviar dados de volta para o cliente em uma requisição HTTP.
- **Endpoint** é um termo em inglês que pode ser traduzido, literalmente, como “pontos de extremidade”. No setor da TI, o termo endpoint pode ser aplicado a diferentes contextos. Em protocolos de comunicação, por exemplo, o endpoint faz referência aos terminais de conexão entre uma API e o cliente.
- **HTTP** (Hypertext Transfer Protocol) é um protocolo de comunicação utilizado para transferir dados pela internet. Ele é a base para a comunicação de dados na World Wide Web (www) e permite que aplicativos cliente-servidor se comuniquem.
- **Integração via API** (Application Programming Interface) é uma forma de relacionar sistemas entre si. Trata-se de uma interface de programação, um recurso muito conhecido por desenvolvedores. Ela ajuda tanto as companhias que criam softwares a disponibilizarem seu produto no mercado, quanto as empresas a implementarem soluções.
- **XML** é a sigla, em inglês, de Extensible Markup Language, ou seja, é uma linguagem de marcação que define um conjunto de regras para codificação de documentos. A linguagem XML é classificada como extensível porque permite definir os elementos de marcação.
- **Token** ou também chamado componente léxico é uma cadeia de caracteres que tem um significado coerente em verdadeira linguagem de programação.
